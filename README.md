# FileStorage

Веб-застосунок для завантаження, перегляду та видалення файлів із лімітом сховища і відстеженням прогресу завантаження.

---

## Стек

- Сервер: Node.js + TypeScript + ESM + `node:http` + бібліотека `file-type`
- Клієнт: React 19 + TypeScript + Tailwind CSS

## Опис

- Завантаження файлів потоком (pipe)
- Відстеження прогресу завантаження
- Перегляд та видалення окремих файлів
- Очищення всього сховища
- Відображення доступного місця у сховищі
- Валідація формату файлу: перевірка розширення + реального mime-типу
- Дозволені формати: `.pdf`, `.png`, `.jpeg`, `.txt`, `.json`
- Ліміт сховища: **3 MB**

## Реалізація

- Кожний ендпоінт обробляє обробник запиту (`server/handlers/...`)
- Для уникнення повторюваного дубльованого коду створено функцію `sendResponse` (`server/utils.ts`), яка приймає необовʼязкові параметри (для універсальності) та будує відповідь серверу із статус кодом, повідомленням та тілом, якщо це передбачено обробником.
- Налаштування CORS - дозволений лише клієнтський оріджин (`http://localhost:3000`), запити інших оріджинів отримують відповідь із помилкою 403 Forbidden.
- Центральний модуль стану сервера (`server/storage.ts`) описує тип метаданих файлу, зберігає всі записи в пам'яті процесу через `Map<string, FileRecord>`, зберігає обʼєкт дозволених mime-типів та реалізує функції повернення сумарного розміру успішно завантажених файлів, резервації місця для файлу до початку запису, і звільнення резерву після завантаження.
- Реалізована санітизація імʼя файлу перед збереженням на диску `normalizeFileName` (`server/utils.ts`)
- Якщо юзер завантажує файл із назвою, яка вже існує, функція `resolveUniqueFileName` обробляє назву і додає унікальне число до назви, таким чином назви файлів ніколи не дублюються.

## API

| Метод    | Маршрут              | Опис                                              |
| -------- | -------------------- | ------------------------------------------------- |
| `POST`   | `/upload`            | Завантажити файл                                  |
| `GET`    | `/upload/status/:id` | Статус завантаження                               |
| `GET`    | `/files`             | Список файлів + інфо про доступне місце у сховищі |
| `GET`    | `/files/:id`         | Отримати файл                                     |
| `DELETE` | `/files/:id`         | Видалити файл                                     |
| `DELETE` | `/files/remove`      | Видалити всі файли                                |

## Запуск

```bash
git clone https://github.com/olyaprykhodko/evoverse
cd evoverse
git checkout http-server
```

### Локальний запуск

1. Сервер (`http://localhost:3500`)

```bash
cd server
cp .env.example .env
npm i
npm start
```

2. Клієнт (`http://localhost:3000`)

```bash
cd client
cp .env.example .env
npm i
npm start
```

### Docker

```bash
docker compose up --build -d
```
